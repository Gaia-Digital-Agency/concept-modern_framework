# CLAUDE.md - Project Context for AI Assistants

## Project Overview

This is a React + Vite + TypeScript + SCSS starter template for branding/marketing websites. It provides a production-ready foundation with responsive design, routing, and testing.

## Tech Stack

- React 18 with TypeScript
- Vite (build tool)
- SCSS with auto-imported mixins
- React Router DOM (routing)
- Vitest + React Testing Library (testing)

## Key Files

| File | Purpose |
|------|---------|
| `src/app.tsx` | Main App component with Router setup |
| `src/main.tsx` | Entry point |
| `src/assets/image.ts` | Centralized image URLs - edit here to change all images |
| `src/styles/main.scss` | Global styles and CSS variables |
| `src/styles/_mixins.scss` | Responsive breakpoint mixins |
| `vite.config.ts` | Vite config with SCSS auto-imports and legacy browser support |
| `tests/app.test.tsx` | Unit tests |

## Architecture Patterns

### Routing
- Uses React Router DOM with `BrowserRouter`
- Routes: `/` (Home), `/about` (About), `/contact` (Contact)
- Navigation in `src/components/navbar.tsx`

### Styling
- SCSS files automatically import `_mixins.scss` via Vite config
- Use `@include respond-to('mobile'|'tablet'|'desktop')` for breakpoints
- Use `@include portrait` or `@include landscape` for orientation
- Breakpoints: mobile < 768px, tablet 768-1023px, desktop >= 1024px

### Components
- `Navbar` - Responsive nav with mobile hamburger menu and dropdowns
- `Footer` - Analytics-ready with geolocation placeholder
- Pages in `src/pages/` directory

## Common Tasks

### Adding a new page
1. Create component in `src/pages/`
2. Add route in `src/app.tsx` inside `<Routes>`
3. Add navigation link in navbar

### Changing images
Edit `src/assets/image.ts` - all components reference this file

### Adding styles
- Global styles go in `src/styles/main.scss`
- Mixins available automatically in all SCSS files

### Running tests
```bash
npm run test
```

## Development Commands

```bash
npm install    # Install dependencies
npm run dev    # Start dev server
npm run test   # Run tests
npm run build  # Production build
```

## Notes

- No external state management (uses React hooks)
- No package.json in repo - needs to be created with dependencies
- Legacy browser support configured for 99% coverage
